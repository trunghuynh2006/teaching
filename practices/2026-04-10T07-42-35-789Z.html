When a one-to-many relationship is created between two tables, a lookup column is created on the related table on the many sides of the relationship. This column links the parent record with the child record. A lookup column is also known as the foreign key. Alternate keys are created for integrations with other systems. Customer is a special lookup column that represents an account or a contact. You cannot create a customer column by creating a one-to-many relationship. In Microsoft Dataverse, every table has a primary key that uniquely identifies each row in the table. The primary key column is created when the table is created.
