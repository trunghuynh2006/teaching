You have a Microsoft Power Apps canvas app named App1 that records guest check-in details at a hotel. App1 uses a Microsoft Dataverse table.

You need to design App1 to meet the following requirements:

  • Collect the guest’s name, meal choice, and additional preferences.
  • Display a message if a guest selects a meal option that is unavailable.

What should you do?

Select only one answer.

The best solution is to add an Edit form bound to the Dataverse table, include the guest’s name, meal choice, and preferences as required fields, and use a formula on the meal choice control (such as an If statement) to check for unavailable options and display a message. This ensures the validation happens immediately when the user makes a selection, which improves usability. Creating a new app is unnecessary because App1 already exists. Adding or renaming a column and marking the Name field as required does not address the requirement of showing a conditional message. Using the OnSubmit property would only display the message after the user submits the form, not when the meal option is selected, which does not meet the requirement for real-time feedback.
